LLM Migration Assistant — SQL → MongoDB

A portable HTML version of the interview-ready answer: architecture, prompts, code snippets, and the one-paragraph summary — all in a consistent LLM-styled design.

Story Scope — An internal Spring Boot dev tool with two endpoints: /migrate-query and /validate.

This single-page tool is intended for developers during a SQL → MongoDB migration. It uses a deterministic LLM setup (low temperature, JSON-only system prompts) to convert SQL queries to MongoDB aggregation pipelines and to validate migrated documents vs source rows.

Architecture Overview

Developer
   │
   ├── POST /migrate-query  (SQL string)
   │        └── LLM API → returns MongoDB aggregation JSON
   │
   └── POST /validate       (mismatched docs: SQL row + MongoDB doc)
            └── LLM API → returns reason + fix suggestion

Two endpoints, one LlmApiClient, and one controller — simple, focused, and practical.

Step 1 — LLM API Client Wrapper

Key design choices: low temperature for deterministic code, system prompt to enforce JSON-only responses, and a reusable @Service for all LLM calls.

@Service
public class LlmApiClient {

    private final RestTemplate restTemplate;

    @Value("${llm.api.url}")
    private String apiUrl;

    @Value("${llm.api.key}")
    private String apiKey;

    @Value("${llm.model}")
    private String model;

    public LlmApiClient(RestTemplateBuilder builder) {
        this.restTemplate = builder.build();
    }

    public String complete(String systemPrompt, String userMessage) {
        HttpHeaders headers = new HttpHeaders();
        headers.setContentType(MediaType.APPLICATION_JSON);
        headers.setBearerAuth(apiKey);

        Map<String, Object> body = Map.of(
            "model", model,
            "messages", List.of(
                Map.of("role", "system",  "content", systemPrompt),
                Map.of("role", "user",    "content", userMessage)
            ),
            "temperature", 0.2
        );

        HttpEntity<Map<String,Object>> request = new HttpEntity<>(body, headers);
        Map<String, Object> response = restTemplate.postForObject(apiUrl, request, Map.class);

        List<Map<String, Object>> choices = (List<Map<String, Object>>) response.get("choices");
        Map<String, Object> message = (Map<String, Object>) choices.get(0).get("message");
        return (String) message.get("content");
    }
}

Step 3 — MigrationToolService (Core Logic)

Two features implemented here: Query translation and migration validation. Both call the same LLM client with strict system prompts asking for JSON-only responses.

@Service
public class MigrationToolService {

    private final LlmApiClient llmApiClient;
    private final ObjectMapper objectMapper;

    public MigrationToolService(LlmApiClient llmApiClient, ObjectMapper objectMapper) {
        this.llmApiClient = llmApiClient;
        this.objectMapper = objectMapper;
    }

    public MigrateQueryResponse migrateQuery(MigrateQueryRequest request) {

        String systemPrompt = """
            You are a database migration expert.
            Convert SQL or JPQL queries into MongoDB aggregation pipelines.
            Always return a valid JSON object with two fields:
            - "aggregationPipeline": the MongoDB pipeline as a JSON array
            - "explanation": a brief explanation
            Return ONLY the JSON object, no markdown, no extra text.
            """;

        String userMessage = String.format("Target MongoDB collection: %s\nSQL Query:\n%s",
            request.getCollectionName(), request.getSqlQuery());

        String rawResponse = llmApiClient.complete(systemPrompt, userMessage);

        try {
            JsonNode json = objectMapper.readTree(rawResponse);
            MigrateQueryResponse response = new MigrateQueryResponse();
            response.setAggregationPipeline(json.get("aggregationPipeline").toString());
            response.setExplanation(json.get("explanation").asText());
            return response;
        } catch (Exception e) {
            MigrateQueryResponse response = new MigrateQueryResponse();
            response.setAggregationPipeline(rawResponse);
            response.setExplanation("Raw response — JSON parsing failed");
            return response;
        }
    }

    public ValidateResponse validate(ValidateRequest request) {

        String systemPrompt = """
            You are a data migration validation expert.
            Given an original SQL record and its migrated MongoDB document,
            return a JSON object with "likelyCause", "suggestedFix", and "severity".
            Return ONLY the JSON object, no markdown, no extra text.
            """;

        String userMessage = String.format("Table: %s → Collection: %s\n\nOriginal SQL record:\n%s\n\nMigrated MongoDB document:\n%s",
            request.getTableName(), request.getCollectionName(), request.getSqlRow(), request.getMongoDocument());

        String rawResponse = llmApiClient.complete(systemPrompt, userMessage);

        try {
            JsonNode json = objectMapper.readTree(rawResponse);
            ValidateResponse response = new ValidateResponse();
            response.setLikelyCause(json.get("likelyCause").asText());
            response.setSuggestedFix(json.get("suggestedFix").asText());
            response.setSeverity(json.get("severity").asText());
            return response;
        } catch (Exception e) {
            ValidateResponse response = new ValidateResponse();
            response.setLikelyCause(rawResponse);
            response.setSuggestedFix("Manual review needed");
            response.setSeverity("UNKNOWN");
            return response;
        }
    }
}

Step 4 — Controller

@RestController
@RequestMapping("/api/migration-tool")
public class MigrationToolController {

    private final MigrationToolService migrationToolService;

    public MigrationToolController(MigrationToolService migrationToolService) {
        this.migrationToolService = migrationToolService;
    }

    @PostMapping("/migrate-query")
    public ResponseEntity<MigrateQueryResponse> migrateQuery(@RequestBody MigrateQueryRequest request) {
        return ResponseEntity.ok(migrationToolService.migrateQuery(request));
    }

    @PostMapping("/validate")
    public ResponseEntity<ValidateResponse> validate(@RequestBody ValidateRequest request) {
        return ResponseEntity.ok(migrationToolService.validate(request));
    }
}

One-paragraph Interview Answer

"We built an internal Spring Boot dev tool with two endpoints. The first took a SQL or JPQL query, called the LLM API with a strict system prompt to return only JSON, and gave back the MongoDB aggregation pipeline equivalent with an explanation. The second was for our validation phase — our batch job flagged mismatches between the MySQL source and MongoDB target, and developers could POST those two documents to the validate endpoint. The LLM would compare them and return the likely cause, a suggested fix, and a severity rating. The key engineering decisions were keeping temperature low for deterministic output, enforcing JSON-only responses in the system prompt, and having a fallback handler when the LLM returned malformed output."
